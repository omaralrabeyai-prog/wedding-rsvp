📋 RSVP Setup Guide

7 visual steps to connect your wedding site to Google Sheets

1 Create a Google Sheet

Open your browser and go to sheets.new — this automatically creates a blank spreadsheet.

📍 What you'll see in the URL bar:
https://docs.google.com/spreadsheets/d/1aBcDeFgHiJkLmNoPqRsTuVwXyZ123456789/edit

The long string between /d/ and /edit is your Spreadsheet ID. → Copy this now, you'll need it in Step 3.

💡 Tip: Rename your sheet tab (bottom-left) from "Sheet1" to RSVPs so it matches the code.
2 Open the Apps Script Editor

Inside your Google Sheet, click the Extensions menu, then Apps Script.

🧭 The menu path:
Extensions → Apps Script

This opens a new browser tab with the Apps Script editor. It starts with a default empty function.

3 Paste the Code & Add Your Sheet ID

Delete the default function myFunction() { } that appears. Open the apps-script.gs file from the project folder and paste the entire contents.

✏️ IMPORTANT: Find this line in the code and replace YOUR_SPREADSHEET_ID_HERE with the ID you copied in Step 1.
// Replace this with your Google Sheet's ID
const SPREADSHEET_ID = '1aBcDeFgHiJkLmNoPqRsTuVwXyZ123456789';

(Don't worry — the sheet ID isn't secret. It's the same ID that's already in the URL of your sheet.)

4 Save the Project

Click the Save icon (💾) or press Cmd+S (Mac) / Ctrl+S (Windows).

📁 Name your project:
Project title (top-left of editor):
Wedding RSVP Backend

Call it whatever you want — this is just for your own reference.

5 Deploy as a Web App

This is the most important step. Click the Deploy button, then New Deployment.

⚙️ The deployment settings:
① Click the dropdown:
Select type → Web app
② Execute as:
Me (your email) ✓ correct
③ Who has access:
Anyone ✓ correct
⚠️ When you click "Deploy", Google will ask you to review permissions. It'll say "This app isn't verified" — this is normal for personal projects. Click "Review Permissions" → choose your Google account → click "Advanced" → "Go to [Project Name] (unsafe)" → "Allow".

Why does it say "unsafe"? Because the app isn't reviewed by Google. You wrote the code yourself — it only writes RSVPs to your own sheet. It's safe.
6 Copy Your Web App URL

After deploying, a dialog box appears with your Web App URL. It looks like this:

https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/exec
💡 Copy this URL now! If you close the dialog, you won't see it again. (You can find it later via Deploy → Manage Deployments, but it's easier to copy now.)
7 Connect the URL to Your Website

Open index.html from the project folder in a text editor (like TextEdit or VS Code). Find this line and paste your URL:

// Change this line (around line 247):
const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';

After that:

  • Test it: Open index.html in your browser, fill out the form, submit.
  • Check your sheet: Go back to your Google Sheet — you should see a new row with the data.
  • Host it: Drag the wedding-rsvp folder to Netlify Drop for a free, instant URL.
✓ Done — You're Live!

That's it. Every time a guest submits the RSVP form, a new row appears in your Google Sheet. No database, no server, no monthly fees.

You can check RSVPs anytime by just opening your Google Sheet — no tech skills needed.
